# Self-Operating Computer — Client-Server AI Agent 🖥️🤖

Кастомизированный форк opensource-проекта Self-Operating Computer, переведенный на эффективную клиент-серверную архитектуру через асинхронные веб-сокеты (Socket.IO). ⚡

В отличие от оригинальной архитектуры, где ИИ-модель и OCR-движок нагружали целевую машину, данный проект разделяет обязанности:
1. Управляющий сервер: Исполняет логику инференса мультимодальной модели gemini-2.5-flash и распознавания текста через easyocr. 🧠
2. Легковесный агент (main.py на клиенте): Работает на целевом компьютере, транслирует скриншоты по запросу и исполняет нативные низкоуровневые команды ОС (pyautogui). 👾

## Архитектура системы и жизненный цикл сессии ⚙️

### Компоненты:
* Управляющий сервер: Инференс модели Gemini 2.5, движок EasyOCR, ASGI/Uvicorn сервер (порт 9003). 🌐
* Локальный агент: Эмулятор команд PyAutoGUI, перехватчик клавиш pynput. ⌨️

### Workflow: 🔄
1. Handshake: Клиент подключается к серверу и передает тип ОС (Windows/Darwin/Linux), целевую задачу (objective) и идентификатор модели через событие client_info.
2. Сигнал Старта: Сервер переводит сессию в статус запуска и отправляет клиенту триггер start_task.
3. Запрос Контекста: Сервер запрашивает снимок экрана (request_screenshot), клиент асинхронно генерирует PNG-байты и возвращает их.
4. Поиск координат: Сервер сохраняет скриншот, отправляет его в gemini-2.5-flash, парсит JSON-ответ, дополняет его точными координатами из easyocr (для операции click) и возвращает клиенту.
5. Выполнение: Клиент выполняет пачку операций и запрашивает следующий шаг.

## Особенности реализации и работы с Промптами 📝

* Управление промптами на клиенте: Все системные промпты для моделей (Standard, Labeled, OCR) и логика формирования контекста на стороне клиента настраиваются в файле client/operate/models/prompts.py. Если вам нужно изменить поведение агента, скорректировать инструкции или добавить новые правила для ИИ — редактируйте этот файл. ✍️
* Локализация и маппинг клавиатуры: В prompts.py интегрирована кастомная логика для работы с русскоязычными операционными системами. Так как эмуляторы ввода имеют ограничения, ИИ-модель обучена на лету переводить русские команды в соответствующие символы английской раскладки (например, при необходимости открыть "Проводник", модель отправляет строку "Ghjdjlybr"). 🇷🇺 ➡️ 🇺🇸
* Кроссплатформенные макросы: Метод get_system_prompt автоматически адаптирует горячие клавиши сворачивания, разворачивания окон и поиска приложений под текущую ОС клиента (Windows, macOS, Linux). 🪟🍏🐧
* Оптимизация OCR: Сервер инициализирует easyocr.Reader один раз за сессию и делает искусственную задержку в 3 секунды перед чтением экрана, давая интерфейсу ОС время на отрисовку анимаций. 🔍

## Инструкция по запуску 🚀

### 1. Настройка переменных окружения (.env) 🔑
Перед запуском сервера создайте в корневой директории файл .env и добавьте туда свой API-ключ:
GOOGLE_API_KEY="ваш_api_ключ_gemini"

### 2. Как поднять серверную часть в Docker 🐳
Сервер принимает подключения от клиентов, обрабатывает OCR и общается с API Gemini. Из-за специфичных зависимостей под Linux и CUDA (библиотеки torch зафиксированы под cu121), серверная часть разворачивается строго в Docker-контейнере.

В корневой директории сервера подготовлен файл docker-compose.yml, настроенный для проброса ядер GPU NVIDIA внутрь изолированной среды. Чтобы собрать образ и поднять сервер на порту 9003, выполните команду:
docker compose up --build -d 🔥

### 3. Запуск клиентского агента 🏃‍♂️
На машине, которой необходимо управлять, перейдите в подпапку исполнительного модуля клиента, установите локальные зависимости и запустите агент:
Команды:
cd client/operate
pip install -r ../requirements.txt
python main.py

По умолчанию в main.py зашита задача автоматического поиска и запуска приложения Postman со сворачиванием всех лишних окон.

### 4. Автономный CLI-режим 🎛️
Вы также можете использовать оригинальный консольный интерфейс для локального выполнения задач без веб-сокетов с помощью зарегистрированной команды (доступно после pip install -e . внутри папки client):
Команда: operate --model gemini-pro-with-ocr --prompt "Open Terminal and clear history"

## Безопасность и Экстренная Остановка (Fail-Safe) 🚨

Поскольку ИИ-агент обладает полным доступом к мыши и клавиатуре, в коде реализовано два независимых механизма прерывания:
1. Клавиша ESC (Рекомендуемый): Внутри клиента работает фоновый поток pynput.keyboard.Listener. Нажатие на клавишу ESC мгновенно выставляет флаг config.stop_execution_flag = True, останавливая отправку команд. 🛑
2. Заглушка углов (PyAutoGUI FailSafe): Если агент заблокировал ввод, физически переведите курсор мыши в любой крайний угол экрана. pyautogui выбросит исключение и аварийно завершит процесс. 🧭
